


在思科（Cisco）等主流网络设备中，当同一台路由器上同时配置了静态 NAT（Static NAT）**和**动态 NAT（Dynamic NAT / PAT）时，它们的匹配和处理优先级遵循非常明确的原则。

简单来说：**静态 NAT 的优先级绝对高于动态 NAT。**

以下是详细的逻辑总结、数据流向分析以及应用场景：

## 1. 核心结论：为什么静态 NAT 优先？

- **匹配机制不同**：
    
    - **静态 NAT** 是一对一（1:1）**的固定映射，在路由器的 NAT 表项中是**永久存在（Static Entry）的。
        
    - **动态 NAT/PAT** 是基于流量触发的，只有当数据包匹配了 ACL 且通过特定算法（如轮询或端口复用）时，才会**动态生成临时表项（Dynamic Entry）**。
        
- **路由器的查找顺序**：当一个数据包进入进行 NAT 转换时，路由器会**优先查找静态 NAT 表项**。如果匹配成功，直接进行转换；只有在静态 NAT 完全没有匹配到的情况下，数据包才会去匹配动态 NAT 的 ACL 策略。
    

## 2. 不同流量方向的优先级表现

我们可以从“由内向外”和“由外向内”两个方向来观察它们的行为：

### 内部发起连接（Inside -> Outside）

假设内部主机 $A$（IP：`192.168.1.10`）同时满足以下两个条件：

1. 配置了静态 NAT 映射到公网 IP `202.1.1.10`。
    
2. 它的 IP 被包含在动态 NAT/PAT 的 ACL 范围内（使用公网 IP 池或接口地址）。
    

- **实际行为**：当主机 $A$ 访问外网时，路由器直接命中静态 NAT 表项，将其源地址转换为 `202.1.1.10`。**动态 NAT 的策略直接被跳过**，不会为其分配动态 IP 或转换端口（PAT）。
    

### 外部发起连接（Outside -> Inside）

- **静态 NAT**：因为表项永久存在，外网用户可以直接通过访问 `202.1.1.10` 来主动发起连接，访问内部的主机 $A$。
    
- **动态 NAT**：外网用户**无法**主动发起连接。因为在没有内部流量触发前，动态 NAT 表项根本不存在。
    

## 3. 常见应用场景与配置避坑

### 场景一：服务器与普通员工混合上网（正确操作）

在一个企业网络中，通常会让普通员工通过 **动态 PAT（Easy IP）** 共享一个公网 IP 上网。同时，内部有一台全天候需要对外提供服务的服务器（如 Web 服务器）。

- **操作**：直接为服务器配置静态 NAT（或者静态 NAT 端口映射 `static pam`），为员工配置动态 PAT。
    
- **结果**：由于静态 NAT 优先级高，服务器会严格使用固定的公网 IP 进出，而不会混入员工的动态流量池中，保证了业务的稳定性。
    

### 场景二：IP 地址重叠冲突（需要注意的坑）

如果你在配置时，不小心让**动态 NAT 的地址池（Pool）**和**静态 NAT 的公网 IP** 产生了重叠，例如：

- 静态 NAT 占用了 `202.1.1.10`
    
- 动态 NAT 地址池范围是 `202.1.1.5` 到 `202.1.1.15`（包含了 `.10`）
    
- **潜在风险**：虽然内部发起流量时静态 NAT 优先，但如果是其他内部主机触发了动态 NAT，路由器有可能会把重叠的 `202.1.1.10` 动态分配给其他主机。这会导致**地址冲突**，造成静态 NAT 的服务器或者动态上网的用户网络中断。
    
- **最佳实践**：**动态 NAT 的地址池必须严格排除掉静态 NAT 已经使用的 IP 地址。**
    

## 总结顺口溜

> **静态永远大于动，固定映射最优先；**
> 
> **内网外发先查静，外网内访找静态；**
> 
> **配置池子要隔离，地址重叠出大问题。**
